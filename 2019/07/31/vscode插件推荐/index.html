前言

从sublimeText转vscode有一段时间了。刚开始时候vscode还有点不习惯，现在觉得超级好用呀！在这里我来推荐一些我觉得好用的插件，让你开发更加高效。事半功倍，爱上开发。哈哈

vscode怎么安装插件

既然是推荐vscode插件，首先不得不提下vscode怎么去安装插件。

（1）打开vscode编辑器

（2）点击右侧拓展图标

（3）搜索需要插件

这个就是目前我使用编辑器主体，如果你不喜欢这个风格，可以去vscode插件系统搜索你喜欢的风格。目前vscode存在2000千多种主体风格

主体传送门

（2）字体

挑选到了喜欢的主题风格之后，那么接下来的工作就是挑选一款心仪的字体。。超级重要，喜欢的主题风格和字体可以让你开心编码。美好的东西大家都喜欢。

windows系统中vscode的字体使用顺序是：consolas, ‘Microsoft YaHei’, monospace

在这里我使用的是Monaco字体，也可以选择Source Code Pro字体。默认情况下我们电脑中是不存在这2中字体的。我们可以在电脑C:\Windows\Fonts盘看看系统中存在字体文件。

如果电脑不存在Monaco字体：

（1）网上下载Monaco字体

（2）把下载字体放在C:\Windows\Font中

（3）设置编辑器的字体：settings.json文件，”editor.fontFamily”:”Monaco”

（4）重启编辑器。

（3）图标

VSCode Great Icons：根据不同文件类型，给文件加上不同图标，支持100+文件类型。这样我们在查看长长的文件列表的时候，可以直接通过文件的图标就可以快速知道文件的类型，而不是去看文件的后缀。

插件地址：VScode Great Icons

代码检测

说到代码检测就不得你提大名鼎鼎的Eslint,Eslint是一个用来识别 ECMAScript 并且按照规则给出报告的代码检测工具。我们用编辑器上安装eslint集成，可以在编写代码时，就会出现错误或者警告提示，不用每次执行eslint 来知道是否有问题。

（1）eslint

安装eslintc插件之后，它会默认识别我们项目根目录下的eslint配置。并在编辑器上给出相应的提示。
