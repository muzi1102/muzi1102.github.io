记录一些git相关的操作

发表于 2016-06-05   |   分类于 git

前言

说实话，使用git有一段时间了(/(ㄒoㄒ)/~~)。每次用就是那几个命令(push,add,commit,clone)。哎!平时要是要用到其他命令，就去网上查。只对几个命令记得熟。伤感。今天就来写一写我记不太住的命令。晕啊。记性好差劲。(ps:对git命令不熟的可以在命令行敲 git help)

—————–这里是分割线—————————————————

删除远程的分支

昨天，整一个东西，不小心搞错的。把一个分支上传到了远程。怎么删除？？不知道啊！那就查吧！
看到一条:

git branch -d branch-name

病急瞎投医。不管三七二十一。先试一试。不成功，发现只是删除的本地对该远程分支的track
那正确的方法是什么：

git push origin :branch-name

ps:冒号前面的空格不能少，原理是把一个空分支push到server上，相当于删除该分支(网上这么说)

删除文件

不小心把不需要的文件上传的远程了。怎么办？简单。删除撒。这真是一句废话啊。嘻嘻！在git中我们可以通过git rm命令把一个文件删除

git rm -rf dir
git add -A
git commit -m "remove dir"
git push origin master

使用git page

话说，这个东西，简直太棒了。好吗？我的博客用的就是git pages+hexo搭建的。当初糊里糊涂。按照网上的教程来。那个时候对node啥的。一点都不了解。今天说的是利用git pages展示个人的小demo。好像是一个账号只有一个git pages。但是创建仓库。可使用多个git pages。

+ 1创建一个repo
+ git clone 仓库 url
+ 创建一个gh-pages分支。git checkout --orphan gh-pages
+ 最后git add,commit,push即可

ps:git checkout的时候后，分支会包含master分支的内容。不想要删除即可。详情请戳https://help.github.com/articles/creating-project-pages-manually/
